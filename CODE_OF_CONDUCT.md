# Code of Conduct

This project is maintained as an open-source learning and reference repository. Contributors are expected to keep discussions professional, constructive, and focused on the work.

## Expected behavior

- Be respectful and concise in issues, pull requests, and reviews.
- Critique code, documentation, and decisions rather than people.
- Assume good intent, but ask for clarification when something is unclear.
- Keep feedback actionable and tied to the project scope.
- Respect the repository's focus on reproducibility, validation, and transparent limitations.

## Unacceptable behavior

- Harassment, insults, or personal attacks.
- Discriminatory or exclusionary language.
- Spam, off-topic promotion, or repeated low-quality comments.
- Sharing private information without permission.
- Pushing unreviewed or unverifiable claims about model quality, production readiness, or business impact.

## Enforcement

The maintainer may edit, hide, or remove comments and may close issues or pull requests that do not follow this code of conduct. Repeated violations may result in being blocked from participating in the repository.

## Reporting

If you see behavior that violates this code of conduct, contact the maintainer through the available GitHub profile or repository discussion channels. Provide a concise description and relevant links so the issue can be reviewed.
